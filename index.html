Lab 01

UI Layouts

Built the foundational screen structure of LexiPath using Jetpack Compose. Explored Column, Row, Box, and LazyColumn to create the Home screen and vocabulary list — the first look at Bobo's world.

Reflection: Declarative UI clicked for me when I stopped thinking in XML. Seeing composables rebuild themselves automatically when state changes made me appreciate how Compose simplifies complex layouts for a language learning app.
▶ Watch on YouTube
Lab 02

User Interaction

Added buttons, text fields, and click handlers to LexiPath. Implemented state using remember {} and mutableStateOf — letting learners type words, trigger quizzes, and see instant feedback.

Reflection: State management is the heart of interactive learning apps. Once I understood recomposition, I could see exactly how a learner's tap should trigger a quiz answer check — clean, predictable, and fast.
▶ Watch on YouTube
Lab 03

Material Design 3

Applied Material 3 theming to unify LexiPath's visual identity — TopAppBar, Cards, FAB, NavigationBar, and a custom color scheme that matches Bobo's warm, earthy personality.

Reflection: Centralising the whole app's look inside one MaterialTheme felt like proper engineering. Typography, color, and shape becoming consistent across all screens gave LexiPath a polished, trustworthy feel for learners.
▶ Watch on YouTube
Lab 04

Navigation & ViewModel

Integrated NavController and NavHost to connect all of LexiPath's screens. Shared a single ViewModel across routes so vocab progress and quiz scores persist as users navigate between screens.

Reflection: The moment NavHost clicked — that it's just a composable swapping content — the whole Compose mental model snapped into place. Clean navigation is what makes a learning app feel smooth and non-frustrating.
▶ Watch on YouTube
Lab 05

Room Database

Built local persistence for LexiPath's saved vocabulary using Room. Defined @Entity, @Dao, and @Database, then observed saved words as StateFlow — so learners never lose their word collections offline.

Reflection: Room's Flow-based observation pattern is perfect for reactive vocabulary lists. Data updates automatically flow to the UI without manual refresh — critical for a learning app where users build word banks over time.

Project 01

LexiPath v1.0

Problem Statement: Many students lack access to engaging vocabulary learning tools. Existing apps are expensive, ad-heavy, or require constant internet — leaving quality education inaccessible for those with limited resources.

The first version of LexiPath introduced a 5-screen app with Jetpack Compose and Navigation. Users can browse vocabulary lists, take quizzes, track progress, and adjust settings — all guided by Bobo the capybara as a friendly learning companion.

Lessons Learned: Sharing state cleanly through a ViewModel across multiple screens taught me why separation of concerns matters. The app felt alive and consistent the moment data stopped living inside individual composables.

Project 02

LexiPath v2.0

Problem Statement: Static word lists don't reflect real-world language richness. Learners need live word relationships, a shareable community space, and interactive challenges to stay motivated and retain vocabulary effectively.

LexiPath v2 is a fully connected, persistent, and hardware-aware upgrade. The new Word Explorer fetches live synonyms and related words from the Datamuse API. Community Wall lets users share discoveries via Firebase Firestore. Shake the phone to trigger a surprise vocabulary challenge via the accelerometer. All vocabulary saves locally with Room — accessible offline, always.

🗄️ Room DB — offline vocab
☁️ Firebase — Community Wall
🌐 Datamuse API — live words
📳 Accelerometer — Shake to Challenge
Lessons Learned: Connecting the accelerometer's SensorEventListener to Compose recomposition required careful lifecycle management. Once it worked, "Shake to Challenge" felt like magic — and I finally understood why hardware integration makes learning genuinely engaging.

Reflection & Learning Journey

When I started this course, I had never written a single line of Kotlin. Building LexiPath from a blank Android Studio project to a cloud-connected, sensor-integrated vocabulary app has been one of the most challenging and rewarding experiences of my studies. Every lab pushed me a little further — from composables to ViewModel, from navigation to Room, and finally to Firebase and hardware sensors.

Choosing SDG 4 (Quality Education) as my theme gave every technical decision a real purpose. Every feature — the Datamuse API's live word data, the offline-capable Room database, the community-driven Firestore Wall — exists because language learners deserve tools that are rich, reliable, and accessible. Bobo the capybara wasn't just a mascot; he reminded me throughout that learning should feel warm, approachable, and never intimidating.

This semester has prepared me to think like a real mobile developer: considering architecture, data flow, user experience, and hardware capabilities together. I walk away confident in Jetpack Compose, MVVM patterns, and API integration — and genuinely excited to keep building.
